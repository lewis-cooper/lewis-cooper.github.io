    Recommended

    Install the latest macOS build

    • Download TodoTerminal-macOS-latest.dmg directly from this site.
    • Open the DMG and drag Todo Terminal.app into Applications.
    • For the first launch, right-click the app and choose Open.
    • If macOS still blocks it, open System Settings > Privacy & Security and click Open Anyway.
    Download Mac App

    This beta DMG is ad hoc signed for structural validity, but it is still not Developer ID signed or notarized.

    Build it yourself

    Use Xcode if you prefer the local source path

    • Open TodoTerminal.xcodeproj and select the Todo Terminal scheme.
    • Run bash build_and_package.sh to build, sign, and package the beta DMG locally.
    • Use ./scripts/package_app.sh whenever you want to refresh the latest website DMG and checksum.

    If you later join the Apple Developer Program, the repo still includes ./scripts/package_release.sh for a signed, notarized release flow.

    Storage path

    Lists and app preferences are stored locally in Application Support, and Settings can reveal that folder in Finder.

    ~/Library/Application Support/TodoTerminal

    Migration built in

    Older data from previous storage locations is migrated automatically on first run when Todo Terminal finds it.

    Release-ready repo

    The repository includes build, test, and packaging scripts for publishing the latest website DMG straight from docs/downloads/.

    FAQ

    Short answers to the practical bits.

    Is Todo Terminal macOS-only?

    Yes. The native app targets macOS 14 and newer, and this page is intentionally focused on the macOS release path.

    Do you store my data?

    No user data is sent to us. Todo lists are stored locally on your Mac in a folder you control, and the app does not require an account or cloud backend.

    Does the app include widget support?

    The widget target is still in the source tree for properly signed development builds, but the website beta DMG ships the standalone app only.

    Why does macOS warn on first launch?

    The website download is not Developer ID signed or notarized. After you drag the app into Applications, first-launch it with Right-click > Open. If macOS still blocks it, go to System Settings > Privacy & Security and click Open Anyway.
